# Training Project - Phase 01 / Proyecto Formativo - Fase 01

## English

### Project Structure

This directory contains the complete structure for all phases of the training project, organized by project activities following the software development lifecycle.

### Phases and Activities

#### 📊 01-analysis (Analysis Phase)
Analysis phase focused on understanding requirements and designing the solution.

- **01-induction**: Introduction to the project and initial setup
- **02-requirements**: Requirements gathering and documentation
- **03-conceptual-model**: Conceptual modeling and design
- **04-algorithms**: Algorithm design and pseudocode

#### 📋 02-planning (Planning Phase)
Planning phase for architecture and technical design.

- **05-prototype**: Prototype development and validation
- **06-architecture**: System architecture design
- **07-technical-proposal**: Technical proposal and documentation

#### ⚙️ 03-execution (Execution Phase)
Implementation and development phase.

- **08-development**: Core development and coding
- **09-database-ui**: Database design and user interface implementation
- **10-deployment**: Deployment and production setup

#### ✅ 04-evaluation (Evaluation Phase)
Testing and evaluation phase.

- **11-testing**: Testing and quality assurance
- **12-evaluation**: Final evaluation and documentation

### Activity Structure

Each activity follows a standardized structure with four subdirectories:

1. **01-learning-guide**: Learning guides and instructions
2. **02-learning-activity**: Activities and exercises
3. **03-support-material**: Reference materials and resources
4. **04-evidence-learning**: Evidence and deliverables

## Español

### Estructura del Proyecto

Este directorio contiene la estructura completa para todas las fases del proyecto formativo, organizado por actividades de proyecto siguiendo el ciclo de vida del desarrollo de software.

### Fases y Actividades

#### 📊 01-analysis (Fase de Análisis)
Fase de análisis enfocada en entender los requisitos y diseñar la solución.

- **01-induction**: Inducción al proyecto y configuración inicial
- **02-requirements**: Recopilación y documentación de requisitos
- **03-conceptual-model**: Modelado conceptual y diseño
- **04-algorithms**: Diseño de algoritmos y pseudocódigo

#### 📋 02-planning (Fase de Planeación)
Fase de planeación para arquitectura y diseño técnico.

- **05-prototype**: Desarrollo de prototipos y validación
- **06-architecture**: Diseño de arquitectura del sistema
- **07-technical-proposal**: Propuesta técnica y documentación

#### ⚙️ 03-execution (Fase de Ejecución)
Fase de implementación y desarrollo.

- **08-development**: Desarrollo principal y codificación
- **09-database-ui**: Diseño de base de datos e implementación de interfaz de usuario
- **10-deployment**: Despliegue y configuración en producción

#### ✅ 04-evaluation (Fase de Evaluación)
Fase de pruebas y evaluación.

- **11-testing**: Pruebas y aseguramiento de calidad
- **12-evaluation**: Evaluación final y documentación

### Estructura de Actividades

Cada actividad sigue una estructura estandarizada con cuatro subdirectorios:

1. **01-learning-guide**: Guías de aprendizaje e instrucciones
2. **02-learning-activity**: Actividades y ejercicios
3. **03-support-material**: Materiales de referencia y recursos
4. **04-evidence-learning**: Evidencias y entregables
