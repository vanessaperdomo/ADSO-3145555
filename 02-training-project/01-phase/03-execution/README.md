# Phase 03: Execution / Fase 03: Ejecución

## English

### Overview

The **Execution Phase** is where the software solution comes to life through coding, database implementation, and UI development. This phase transforms designs into functional, deployable software.

### Objectives

- Develop software modules following market standards
- Implement database structures and user interfaces
- Configure and deploy software systems
- Apply quality assurance practices
- Follow coding best practices and standards

### Activities

This phase contains **3 project activities**:

| # | Activity | Focus Area |
|---|----------|------------|
| 08 | [Development](08-development) | Module coding and implementation |
| 09 | [Database & UI](09-database-ui) | Data structures and interfaces |
| 10 | [Deployment](10-deployment) | Configuration and launch |

### Key Competencies

- **220501096** - Develop software solution according to design and development methodologies
- **240202501** - Interact in English in social and work contexts
- **220501098** - Control software service quality according to technical standards
- **210201501** - Exercise fundamental labor rights within constitutional framework
- **240201529** - Manage entrepreneurial and business culture processes
- **220601501** - Apply environmental protection, safety and health practices

### Learning Outcomes (RAPs)

Key learning outcomes include:
- Plan software construction activities (593106-01)
- Code software according to established design (593108-04)
- Perform software testing to verify functionality (593105-05)
- Create front-end components according to design (593104-03)
- Build database from data model (593107-02)
- Document software implementation process (593109-03)
- Deploy software according to architecture and policies (593110-02)
- Implement software according to service levels (593112-04)

### Deliverables

- Source code (version controlled)
- Functional software modules
- Database implementation
- User interface components
- Unit and integration tests
- Deployment documentation
- Configuration files
- API documentation
- Execution phase report

## Español

### Descripción General

La **Fase de Ejecución** es donde la solución de software cobra vida a través de la codificación, implementación de base de datos y desarrollo de UI. Esta fase transforma los diseños en software funcional y desplegable.

### Objetivos

- Desarrollar módulos de software siguiendo estándares del mercado
- Implementar estructuras de bases de datos e interfaces de usuario
- Configurar y desplegar sistemas de software
- Aplicar prácticas de aseguramiento de calidad
- Seguir mejores prácticas y estándares de codificación

### Actividades

Esta fase contiene **3 actividades de proyecto**:

| # | Actividad | Área de Enfoque |
|---|-----------|-----------------|
| 08 | [Desarrollo](08-development) | Codificación e implementación de módulos |
| 09 | [Base de Datos e IU](09-database-ui) | Estructuras de datos e interfaces |
| 10 | [Despliegue](10-deployment) | Configuración y lanzamiento |

### Competencias Clave

- **220501096** - Desarrollar la solución de software según diseño y metodologías
- **240202501** - Interactuar en lengua inglesa en contextos sociales y laborales
- **220501098** - Controlar calidad del servicio de software según estándares técnicos
- **210201501** - Ejercer derechos fundamentales del trabajo en marco constitucional
- **240201529** - Gestionar procesos de cultura emprendedora y empresarial
- **220601501** - Aplicar prácticas de protección ambiental, seguridad y salud

### Resultados de Aprendizaje (RAPs)

Los resultados de aprendizaje clave incluyen:
- Planear actividades de construcción del software (593106-01)
- Codificar el software según diseño establecido (593108-04)
- Realizar pruebas al software para verificar funcionalidad (593105-05)
- Crear componentes front-end según diseño (593104-03)
- Construir base de datos a partir del modelo de datos (593107-02)
- Documentar proceso de implantación de software (593109-03)
- Desplegar software según arquitectura y políticas (593110-02)
- Implantar software según niveles de servicio (593112-04)

### Entregables

- Código fuente (control de versiones)
- Módulos de software funcionales
- Implementación de base de datos
- Componentes de interfaz de usuario
- Pruebas unitarias e integración
- Documentación de despliegue
- Archivos de configuración
- Documentación de API
- Informe de fase de ejecución
