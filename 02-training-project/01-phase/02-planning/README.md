# Phase 02: Planning / Fase 02: Planeación

## English

### Overview

The **Planning Phase** transforms the analysis results into actionable technical designs and architectural blueprints. This phase focuses on creating prototypes, defining system architecture, and preparing the technical proposal.

### Objectives

- Design software prototypes based on functional requirements
- Define logical and technological architecture
- Structure comprehensive technical proposals
- Establish design patterns and standards

## Español

### Descripción General

La **Fase de Planeación** transforma los resultados del análisis en diseños técnicos accionables y planos arquitectónicos. Esta fase se enfoca en crear prototipos, definir la arquitectura del sistema y preparar la propuesta técnica.

### Objetivos

- Diseñar prototipos de software basados en requisitos funcionales
- Definir arquitectura lógica y tecnológica
- Estructurar propuestas técnicas comprensivas
- Establecer patrones y estándares de diseño

### Actividades

Esta fase contiene **3 actividades de proyecto**:

| # | Actividad | Área de Enfoque |
|---|-----------|-----------------|
| 05 | [Prototipo](05-prototype) | Prototipado funcional y de calidad |
| 06 | [Arquitectura](06-architecture) | Estructura lógica y tecnológica |
| 07 | [Propuesta Técnica](07-technical-proposal) | Documentación del proyecto |

### Competencias Clave

- **220501095** - Diseñar la solución de software según procedimientos y requisitos técnicos
- **240202501** - Interactuar en lengua inglesa en contextos sociales y laborales
- **230101507** - Generar hábitos saludables de vida mediante programas de actividad física
- **240201064** - Orientar investigación formativa según referentes técnicos
- **220501094** - Estructurar propuesta técnica de servicios de TI
- **240201526** - Interactuar en contextos productivos y sociales con principios éticos

### Resultados de Aprendizaje (RAPs)

Los resultados de aprendizaje clave incluyen:
- Determinar características técnicas de la interfaz gráfica (593100-03)
- Estructurar modelo de datos del software según especificaciones (593101-02)
- Verificar entregables de la fase de diseño (593102-04)
- Elaborar artefactos de diseño siguiendo prácticas metodológicas (593103-01)
- Definir especificaciones técnicas del software (593060-01)
- Validar condiciones de la propuesta técnica (593061-03)

### Entregables

- Prototipos interactivos
- Documento de arquitectura del sistema
- Diseño de base de datos (DER, DFD)
- Diagramas de arquitectura técnica
- Propuesta técnica completa
- Documentación de patrones de diseño
- Informe de fase de planeación
