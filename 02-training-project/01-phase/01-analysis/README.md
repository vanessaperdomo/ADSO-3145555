# Phase 01: Analysis / Fase 01: Análisis

## English

### Overview

The **Analysis Phase** is the foundation of the software development lifecycle. This phase focuses on understanding requirements, stakeholder needs, and establishing the conceptual framework for the software solution.

### Objectives

- Contextualize within SENA institutional framework
- Gather and document functional requirements
- Create conceptual models of the solution
- Structure logical processes through algorithms

### Activities

This phase contains **4 project activities**:

| # | Activity | Focus Area |
|---|----------|------------|
| 01 | [Induction](01-induction) | Institutional context and methodology |
| 02 | [Requirements](02-requirements) | Functional specifications |
| 03 | [Conceptual Model](03-conceptual-model) | Software modeling |
| 04 | [Algorithms](04-algorithms) | Logical processes |

### Key Competencies

- **240201530** - Learning outcomes from induction
- **240202501** - Interact in English in social and work contexts
- **220501046** - Use IT tools for information management
- **220501092** - Establish software solution requirements
- **220501093** - Evaluate software solution requirements
- **240201524** - Develop effective communication processes
- **220201501** - Apply knowledge of natural sciences
- **240201528** - Quantitative reasoning in mathematical contexts

### Learning Outcomes (RAPs)

Key learning outcomes include:
- Identify SENA's organizational dynamics (593343-01)
- Collect software information according to client needs (593344-02)
- Model software functions according to requirements (592373-02)
- Develop logical processes through algorithm implementation (592376-03)

### Deliverables

- Induction documentation
- Requirements specification document
- Conceptual models (UML diagrams, ERD)
- Algorithms and pseudocode
- Analysis phase report

## Español

### Descripción General

La **Fase de Análisis** es la base del ciclo de vida del desarrollo de software. Esta fase se enfoca en comprender los requisitos, las necesidades de los interesados y establecer el marco conceptual para la solución de software.

### Objetivos

- Contextualizar dentro del marco institucional del SENA
- Recopilar y documentar requisitos funcionales
- Crear modelos conceptuales de la solución
- Estructurar procesos lógicos a través de algoritmos

### Actividades

Esta fase contiene **4 actividades de proyecto**:

| # | Actividad | Área de Enfoque |
|---|-----------|-----------------|
| 01 | [Inducción](01-induction) | Contexto institucional y metodología |
| 02 | [Requisitos](02-requirements) | Especificaciones funcionales |
| 03 | [Modelo Conceptual](03-conceptual-model) | Modelado de software |
| 04 | [Algoritmos](04-algorithms) | Procesos lógicos |

### Competencias Clave

- **240201530** - Resultados de aprendizaje de la inducción
- **240202501** - Interactuar en lengua inglesa en contextos sociales y laborales
- **220501046** - Utilizar herramientas informáticas para el manejo de información
- **220501092** - Establecer requisitos de la solución de software
- **220501093** - Evaluar requisitos de la solución de software
- **240201524** - Desarrollar procesos de comunicación eficaces
- **220201501** - Aplicar conocimientos de las ciencias naturales
- **240201528** - Razonar cuantitativamente en contextos matemáticos

### Resultados de Aprendizaje (RAPs)

Los resultados de aprendizaje clave incluyen:
- Identificar la dinámica organizacional del SENA (593343-01)
- Recolectar información del software según necesidades del cliente (593344-02)
- Modelar las funciones del software según requisitos (592373-02)
- Desarrollar procesos lógicos a través de implementación de algoritmos (592376-03)

### Entregables

- Documentación de inducción
- Documento de especificación de requisitos
- Modelos conceptuales (diagramas UML, DER)
- Algoritmos y pseudocódigo
- Informe de fase de análisis
