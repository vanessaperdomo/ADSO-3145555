# Activity 02: Requirements / Actividad 02: Requisitos

## English

### Activity Overview

**Determine the functional specifications of the software**

This activity focuses on gathering, documenting, and validating software requirements through stakeholder engagement and systematic analysis of client needs.

### Objectives

- Collect comprehensive information about software to be built
- Document functional and non-functional requirements
- Validate requirements with stakeholders
- Characterize organizational processes
- Establish clear software specifications
- Utilize IT tools for requirements management

### Associated Competencies

| Code | Competency |
|------|------------|
| **240202501** | Interact in English in social and work contexts according to CEFR |
| **220501046** | Use IT tools according to information management needs |
| **220501092** | Establish software solution requirements according to standards and technical procedures |

### Learning Outcomes (RAPs)

| Code | Learning Outcome |
|------|------------------|
| **593117-01** | Understand information about current and future daily and work situations through oral and written social interactions |
| **593151-02** | Apply functionalities of IT tools and services according to user manuals, established procedures, and best practices |
| **593152-04** | Optimize results according to verification |
| **593153-03** | Evaluate results according to requirements |
| **593154-01** | Prepare ICT tools according to information processing and communication needs |

| **593345-04** | Validate requirements report according to client needs |

   - Functional requirements

3. **Validation Reports**

- Clarity and precision of documentation
- Proper use of requirements standards
- Stakeholder validation achieved
- Traceability established
- Appropriate use of IT tools

### Duration

Approximately 3-4 weeks

## Español

### Descripción de la Actividad

**Determinar las especificaciones funcionales del software**

Esta actividad se enfoca en recopilar, documentar y validar los requisitos del software mediante el compromiso con los interesados y análisis sistemático de las necesidades del cliente.

### Objetivos

- Recopilar información completa sobre el software a construir
- Documentar requisitos funcionales y no funcionales
- Validar requisitos con los interesados
- Caracterizar procesos organizacionales
- Establecer especificaciones claras del software
- Utilizar herramientas informáticas para gestión de requisitos
